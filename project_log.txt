## Goal
Compare the Fourier Neural Operator (FNO) and a U-Net (CNN baseline) on predicting
the time evolution of 2D incompressible Navier-Stokes equations. Evaluation metrics:
relative L2 error and inference speed.

## Session 2 — 2026-04-20

### Models Implemented
- dataset.py: NSForcingDataset wrapping the .pt files (x→y vorticity pairs)
- models/fno.py: FNO2d — 4 Fourier layers, modes=12, width=64, ~4.7M params
  - Input augmented with 2D grid coords (x: 3 channels = vorticity + gx + gy)
  - SpectralConv2d truncates to top-12 modes in each dim
- models/unet.py: UNet — 4-level encoder-decoder with skip connections, ~31M params
- train.py: training loop with relative L2 loss, cosine LR, W&B logging
- evaluate.py: reports rel-L2 and inference throughput (samples/sec)

### Verified
- Both models forward/backward pass correctly on CPU
- FNO: ~4.7M params | U-Net: ~31M params

### Dataset Visualization
- visualize_data.py: generates 3 figures saved to figures/
  - figures/sample_pairs.png: 5 examples of input/target pairs. Top row = input
    vorticity fed to the model; bottom row = target it must predict. Shows overall
    data structure.
  - figures/sample_diffs.png: target − input for each sample. Isolates the change
    the model must learn. Diffs are large and structured (not noise), so the model
    cannot just copy the input.
  - figures/stats.png: two summary plots over the full training set:
    - Histogram: input values (blue) are tightly centered; target values (orange)
      spread much wider (~20x more variable).
    - Per-sample std scatter: nearly diagonal — more energetic inputs produce more
      energetic targets. Dataset is consistent and well-behaved.
- Key observations:
  - Inputs are smooth, low-amplitude (std ≈ 0.15, range ≈ ±1.5)
  - Targets are more turbulent, much higher amplitude (range ≈ ±4)
  - Differences are large and structured — model must learn real dynamics
  - Per-sample std is nearly linear: more energetic inputs → more energetic targets
  - Target variance is ~20x wider than input variance; consider normalizing targets

## Next Steps

### Step 1 — Start the MLflow UI (optional but recommended)
  ~/.local/bin/uv run mlflow ui
  Opens dashboard at http://localhost:5000 — shows live loss curves and run comparison.
  MLflow runs locally, no account or API key needed.

### Step 2 — Train FNO
  ~/.local/bin/uv run python train.py --model fno --epochs 100
  - Evaluates on test set every 5 epochs
  - Saves best checkpoint to checkpoints/fno_best.pt
  - Logs train/test rel-L2 to MLflow experiment: comp395-ns-benchmark

### Step 3 — Train U-Net
  ~/.local/bin/uv run python train.py --model unet --epochs 100
  - Same setup, saves to checkpoints/unet_best.pt

### Step 4 — Evaluate both models
  ~/.local/bin/uv run python evaluate.py --model fno  --checkpoint checkpoints/fno_best.pt
  ~/.local/bin/uv run python evaluate.py --model unet --checkpoint checkpoints/unet_best.pt
  - Reports: test rel-L2, throughput (samples/sec), latency (ms/sample)
  - These numbers are the core comparison for the writeup

### Notes
  - Training on CPU will be very slow (hours); GPU recommended
  - PyTorch auto-detects GPU — no code changes needed
  - (Optional) Add target normalization to train.py first to stabilize loss scale
    (targets have ~20x wider variance than inputs)
