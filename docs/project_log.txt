## Training Issues & Fixes — 2026-04-24

### Issue 1: MLflow artifact logging crash
- train.py called mlflow.log_artifact(ckpt_path) after saving the best checkpoint
- On first test evaluation (epoch 5), this crashed with an os.makedirs recursion error
  because MLflow could not create the artifact directory when using a SQLite backend
  without an explicit artifact root configured
- Fix: replaced mlflow.log_artifact() with mlflow.log_param('best_checkpoint', ckpt_path)
  The checkpoint is already saved locally to checkpoints/; recording the path as a param
  is sufficient and avoids the artifact store entirely

### Issue 2: Training frozen on first epoch (DataLoader memory pressure)
- Training process launched but no epoch output printed; process appeared hung
- GPU showed 0% utilization despite the training process holding ~1.6 GB VRAM
- Root cause: num_workers=4 in both DataLoaders caused 4 worker processes to each
  load a copy of the dataset into RAM simultaneously (~2 GB each = ~8 GB total),
  exhausting available RAM and causing the system to swap
- Fix: reduced num_workers from 4 to 2 in both train_loader and test_loader
  This halved the concurrent RAM usage and allowed training to proceed normally

## Dataset Deep Dive — 2026-04-24

### What the dataset contains

Each sample is a pair of 128×128 grids representing vorticity fields:
  - x (input):  vorticity ω at time t   — low amplitude,  std ≈ 0.15, range ≈ [-0.76,  0.82]
  - y (target): vorticity ω at time t+1 — high amplitude, std ≈ 0.71, range ≈ [-3.49,  3.93]

The task is pure supervised regression: given a fluid snapshot, predict the next one.
Loss is relative L2 — scale-invariant, so a random model scores ~1.0 and a good model scores <0.1.

### What the visualisation figures show

figures/sample_pairs.png — 5 columns × 2 rows
  - Top row:    Input #i  — vorticity field fed to the model
  - Bottom row: Target #i — vorticity field the model must predict
  - Both rows share the same color scale per column (RdBu_r: red = positive/clockwise,
    blue = negative/counterclockwise)
  - Inputs look smooth and low-amplitude; targets look more turbulent and spread wider.

figures/sample_diffs.png — 5 plots, one per example
  - Each plot shows: diff = y[i] - x[i]  (target minus input)
  - Isolates the change the model must learn over one timestep
  - Diffs are large and structured (not noise) — the model cannot just copy the input.
    This confirms the task is non-trivial and real physics must be learned.

### What causes the fluid to swirl: Kolmogorov forcing

The dataset name "nsforcing" means Navier-Stokes with forcing. The specific forcing is
Kolmogorov forcing — a fixed sinusoidal body force that continuously pumps energy in at
a low spatial frequency, e.g.:

    f(x, y) = sin(k·y)   (force in x-direction, periodic in y, typically k=4)

Why the fluid ends up swirling (not just flowing straight):
  1. The forcing injects energy at one scale (the forcing wavenumber).
  2. The nonlinear advection term u·∇ω transfers energy to other scales — turbulent cascade.
  3. At ν=1e-4 (Re=10,000), viscous damping is weak, so energy spreads before dissipating.
  4. Result: chaotic, swirling vortex structures — 2D turbulence.

### What fluid it is and what the units are

It is NOT a specific real fluid. The simulation is non-dimensionalized — standard practice
in CFD. All numbers are dimensionless:

  Quantity          In dataset              In reality
  --------          ----------              ---------
  Domain            [0,1]×[0,1] unit sq     some physical area in m²
  Vorticity values  dimensionless ~[-4, 4]  1/seconds in real life
  Viscosity ν       1e-4 (dimensionless)    m²/s for a real fluid
  Time step         1 unit                  some real Δt in seconds

ν=1e-4 gives Re=10,000 (turbulent regime) but does not map to water (ν≈1e-6 m²/s) or
air (ν≈1.5e-5 m²/s) at any specific physical scale. It is a synthetic turbulence benchmark
designed to be hard to predict.

This is intentional: the model learns the dimensionless operator mapping ω(t) → ω(t+1).
Whether that describes water in a pipe or air over a wing depends on how physical scales
are chosen — the dimensionless PDE is the same either way. Li et al. (2021) use this same
non-dimensionalized setup for the same reason.
