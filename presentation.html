What Is Fluid Dynamics?

The study of how liquids and gases move.

🌊 Weather

Air flowing around pressure systems, forming storms and fronts

✈️ Aerodynamics

Air flowing over a wing to generate lift

🩸 Medicine

Blood flowing through arteries and around heart valves


In all these cases, we want to answer: "given the fluid's state right now,
what does it look like a moment later?"

What Is a PDE?

A Partial Differential Equation is an equation that describes how something changes across both space and time at once.


Regular equation

\( x^2 + 3x - 4 = 0 \)

Solve for a single unknown number

PDE

\( \frac{\partial u}{\partial t} = \alpha \frac{\partial^2 u}{\partial x^2} \)

Solve for a function \(u(x,t)\) — a value at every point in space and time


The equation above is the heat equation — how temperature spreads through a rod. Fluid dynamics uses far more complex PDEs.

The Navier-Stokes Equations

These PDEs govern all viscous fluid flow. Written for incompressible 2D flow:

\[ \frac{\partial \mathbf{u}}{\partial t} + (\mathbf{u} \cdot \nabla)\mathbf{u} = -\nabla p + \nu \nabla^2 \mathbf{u} + \mathbf{f} \]

\(\mathbf{u}\)

Velocity field — how fast and which way the fluid moves at every point

\(\nu \nabla^2 \mathbf{u}\)

Viscosity term — how "thick" the fluid is (honey vs water)

\(\mathbf{f}\)

External forcing — e.g. a fan blowing, gravity pulling


We work with vorticity \(\omega = \nabla \times \mathbf{u}\) — a scalar field measuring how much the fluid rotates at each point.

Why Is This Hard to Solve?

Traditional approach

  • Divide space into a fine grid
  • Numerically step forward in time
  • Accurate, but extremely slow

A single 128×128 simulation
can take minutes to hours
on a supercomputer at high fidelity

The turbulence problem

[turbulent vorticity field image]

Tiny changes in initial conditions cascade into wildly different outcomes (chaos)


We need a faster way to simulate fluid dynamics — ideally one that can learn the physics from data.

Neural Operators

A regular neural network maps vectors → vectors.
A neural operator maps functions → functions.


Standard NN

Input: fixed-size vector (e.g. 128 numbers)
Output: fixed-size vector
Tied to one resolution.

Neural Operator

Input: a function over a domain (e.g. a vorticity field at any resolution)
Output: another function
Resolution-independent.


For PDEs this is powerful: learn the operator \(\mathcal{G}: u(t) \mapsto u(t + \Delta t)\) once, then apply it at any resolution.

Fourier Neural Operator (FNO)

Li et al., 2021 — "Fourier Neural Operator for Parametric Partial Differential Equations"


  • Key idea: apply convolutions in frequency space instead of physical space
  • Transform input to frequency space with FFT
  • Keep only the lowest \(k\) Fourier modes (large-scale structure)
  • Apply learned weights, transform back
  • Stack \(N\) of these layers
\[ (Kv)(x) = \mathcal{F}^{-1}\!\bigl(R \cdot \mathcal{F}(v)\bigr)(x) \]

\(\mathcal{F}\) = FFT  ·  \(R\) = learned weights
\(\mathcal{F}^{-1}\) = inverse FFT


Because the operation is defined in frequency space, it works on any input resolution — enabling zero-shot super-resolution.

Each Fourier Layer

  • FFT → truncate to 12 modes
  • Multiply by complex weights
  • Inverse FFT back to spatial
  • Add 1×1 bypass convolution
  • Apply GELU activation

Our FNO config

  • Modes: 12 × 12
  • Width: 64 channels
  • Layers: 4
  • Parameters: ~4.7M

U-Net Baseline

A classic CNN architecture originally designed for image segmentation.
We use it as our convolutional baseline.


  • Encoder: repeatedly halve spatial size, double channels
  • Decoder: upsample back, restoring spatial size
  • Skip connections: paste encoder features directly into decoder to preserve fine detail

No physics-specific design —
it learns purely from data.

Our U-Net config

  • 4 encoder/decoder levels
  • Channels: 64→128→256→512
  • BatchNorm + ReLU
  • Parameters: ~31M

~6.6× more parameters than FNO

Training Setup

Both models

  • Loss: relative L2 error
  • Optimizer: Adam (lr=1e-3)
  • LR schedule: cosine decay
  • Epochs: 100
  • Batch size: 20
  • Hardware: NVIDIA RTX 2060

Relative L2 error

\[ \mathcal{L} = \frac{\|\hat{u} - u\|_2}{\|u\|_2} \]

Measures error as a fraction of the target's magnitude.
Score of 1.0 = random guess.
Score < 0.1 = strong prediction.

Conclusion

What we showed

  • FNOs can learn the Navier-Stokes time-evolution operator from data
  • FNO achieves competitive accuracy with 6× fewer parameters than U-Net
  • FNO generalises to unseen resolutions zero-shot; U-Net [result TBD]

Limitations

  • Single fixed viscosity — no generalisation across ν
  • Single-step prediction only (no rollout)
  • Super-res ground truth is proxy (bilinear upsample)

FNOs represent a new class of model that encodes physical structure (frequency content of PDEs) directly into the architecture — enabling faster, more data-efficient simulation than generic CNNs.
